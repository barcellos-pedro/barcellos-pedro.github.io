# Workspace Prompt: Jekyll Blog Post and Navigation Updates

Use this prompt when you want the AI to generate or update blog content and site navigation for this Jekyll portfolio/blog.

## What it helps with

- Writing a new blog post in `_posts/`
- Updating an existing blog post
- Adding or editing menu entries in `_data/navigation.yml`

## How to use it

Provide a clear request with one of these goals:

- create a new blog post
- revise an existing post
- add or update a navigation item

Include details like:

- title, date, author, and topic for posts
- page URL, label, and menu placement for navigation
- desired tone, audience, or formatting preferences

## Output expectations

- Blog posts should be plain Markdown with short paragraphs, headings, and front matter.
- New posts should use `_posts/YYYY-MM-DD-title.md` naming.
- Navigation changes should keep the existing YAML format and preserve current items.
- Avoid introducing new JavaScript, new build tooling, or layout systems.
- Do not edit generated output in `_site/`.

## Example prompt templates

- "Create a new blog post about learning Jekyll site structure with title `How I Built My Portfolio Blog`, date `2026-05-26`, author `Pedro`, and three short sections."
- "Update the navigation in `_data/navigation.yml` to add a link labeled `Blog` that points to `/blog.html` after `About`."
- "Revise an existing post to make the introduction clearer and add a new heading for key takeaways."
